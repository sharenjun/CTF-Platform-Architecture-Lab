# 题目系统

## 模块定位

题目系统负责 Version 0.1 的题目展示、题目详情和题目基础状态管理。

这个版本不做管理员后台，所以题目可以先通过数据库初始化脚本写入。

## Version 0.1 功能范围

1. 查看题目列表
2. 查看题目详情
3. 展示题目分类
4. 展示题目分值
5. 展示解题人数
6. 展示当前用户是否已解出
7. 为提交 Flag 提供题目入口

## 暂不支持

1. 管理员创建题目
2. 管理员编辑题目
3. 上传附件
4. 题目提示
5. Docker 动态环境
6. 动态分值
7. 题目可见时间
8. 多比赛题目隔离

## 核心实体

### 题目

题目是 CTF 平台的核心资源。

核心字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 题目 ID |
| `title` | 题目标题 |
| `description` | 题目描述 |
| `category` | 题目分类 |
| `score` | 题目分值 |
| `flag_hash` | Flag 哈希 |
| `status` | 题目状态 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

## 题目分类

Version 0.1 可以先用字符串保存分类，不单独建分类表。

建议初始分类：

1. Web
2. Pwn
3. Crypto
4. Reverse
5. Misc

后续如果需要分类排序、分类图标、分类统计，再拆出 `categories` 表。

## 题目状态

| 状态 | 说明 |
| --- | --- |
| `ONLINE` | 上线，可以被用户查看和提交 |
| `OFFLINE` | 下线，不允许普通用户查看和提交 |

Version 0.1 不需要 `DRAFT`，因为没有管理后台。

## Flag 存储规则

Flag 不建议明文存储。

Version 0.1 建议保存 `flag_hash`，提交时对用户输入的 Flag 做相同的校验逻辑。

如果使用 BCrypt：

1. 创建题目时将正确 Flag 用 BCrypt 哈希后保存。
2. 用户提交时使用 `matches(rawFlag, flagHash)` 校验。
3. 数据库中不保存明文 Flag。

## 题目列表展示规则

题目列表只展示 `ONLINE` 题目。

每道题至少返回：

| 字段 | 说明 |
| --- | --- |
| `id` | 题目 ID |
| `title` | 题目标题 |
| `category` | 分类 |
| `score` | 分值 |
| `solved_count` | 解题人数 |
| `solved_by_me` | 当前用户是否已解出 |

列表不返回 `description` 和 `flag_hash`。

## 题目详情展示规则

题目详情只允许查看 `ONLINE` 题目。

题目详情至少返回：

| 字段 | 说明 |
| --- | --- |
| `id` | 题目 ID |
| `title` | 题目标题 |
| `description` | 题目描述 |
| `category` | 分类 |
| `score` | 分值 |
| `solved_count` | 解题人数 |
| `solved_by_me` | 当前用户是否已解出 |

详情接口也不能返回 `flag_hash`。

## 关键异常

| 场景 | 建议错误码 | HTTP 状态 |
| --- | --- | --- |
| 题目不存在 | `CHALLENGE_NOT_FOUND` | 404 |
| 题目已下线 | `CHALLENGE_NOT_FOUND` | 404 |
| 用户未登录 | `UNAUTHORIZED` | 401 |

下线题目对普通用户统一表现为不存在，避免暴露后台数据。
