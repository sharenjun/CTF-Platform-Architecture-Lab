# 判题系统

## 模块定位

判题系统负责接收用户提交的 Flag，判断是否正确，并在正确时写入解题记录。

Version 0.1 只做同步 Flag 判题，不做异步队列、代码运行、自定义判题脚本和 Docker 沙箱。

## Version 0.1 功能范围

1. 接收用户提交的 Flag
2. 创建提交记录
3. 校验题目是否存在且上线
4. 校验用户是否已经解出该题
5. 判断 Flag 正误
6. 正确时写入解题记录
7. 避免重复加分

## 暂不支持

1. 异步判题队列
2. 判题 Worker
3. 判题脚本
4. Docker 沙箱
5. 判题日志文件
6. 提交频率限制
7. 反作弊检测

## 核心实体

### 提交记录

提交记录表示用户对某道题的一次 Flag 提交。

无论正确或错误，都可以记录提交行为。

核心字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 提交记录 ID |
| `user_id` | 用户 ID |
| `challenge_id` | 题目 ID |
| `submitted_flag_hash` | 用户提交内容的哈希摘要 |
| `result` | 提交结果 |
| `created_at` | 提交时间 |

不建议保存用户提交的原始 Flag。

### 解题记录

解题记录表示用户已经成功解出某道题。

核心字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 解题记录 ID |
| `user_id` | 用户 ID |
| `challenge_id` | 题目 ID |
| `score` | 本次解题获得的分数 |
| `solved_at` | 解题时间 |

同一个用户对同一道题只能有一条解题记录。

## 提交结果

| 结果 | 说明 |
| --- | --- |
| `ACCEPTED` | Flag 正确 |
| `WRONG_ANSWER` | Flag 错误 |
| `ALREADY_SOLVED` | 用户已经解出过该题 |

Version 0.1 是同步判题，不需要 `PENDING` 和 `JUDGING`。

## 判题流程

```text
用户提交 Flag
↓
校验用户登录状态
↓
查询题目
↓
确认题目为 ONLINE
↓
判断用户是否已经解出
↓
保存提交记录
↓
校验 Flag
↓
如果正确，写入解题记录
↓
返回判题结果
```

## 重复提交规则

1. 用户未解出题目时，错误提交只记录提交记录，不加分。
2. 用户未解出题目时，正确提交记录提交记录，并创建解题记录。
3. 用户已经解出题目后，再提交正确或错误 Flag，都不重复加分。
4. 是否记录已解出后的重复提交，可以先记录，结果为 `ALREADY_SOLVED`。

## 并发规则

同一用户可能在极短时间内重复提交同一道题。

必须使用数据库唯一约束保证：

```text
solves(user_id, challenge_id) 唯一
```

业务层也要先查询是否已解出，但真正防重复加分必须依赖数据库约束。

## 事务边界

一次提交 Flag 应该在一个事务内完成。

事务内操作：

1. 查询题目
2. 查询是否已解出
3. 保存提交记录
4. 如果正确，保存解题记录

如果保存解题记录时触发唯一约束冲突，说明发生并发重复提交，应返回 `ALREADY_SOLVED` 或按已解出处理。

## 后端建议分层

```text
controller
└── SubmissionController

service
└── SubmissionService

repository
├── ChallengeRepository
├── SubmissionRepository
└── SolveRepository

entity
├── Submission
└── Solve

dto
├── SubmitFlagRequest
└── SubmitFlagResponse
```

## 关键异常

| 场景 | 建议错误码 | HTTP 状态 |
| --- | --- | --- |
| 用户未登录 | `UNAUTHORIZED` | 401 |
| 题目不存在 | `CHALLENGE_NOT_FOUND` | 404 |
| Flag 为空 | `VALIDATION_ERROR` | 400 |
| 系统判题异常 | `SUBMISSION_FAILED` | 500 |
