# 排行榜系统

## 模块定位

排行榜系统负责根据解题记录计算用户排名。

Version 0.1 不引入 Redis 缓存，先使用数据库实时查询计算排行榜。

## Version 0.1 功能范围

1. 查询用户总分
2. 查询用户解题数量
3. 查询用户最后一次正确提交时间
4. 按规则生成排名
5. 返回排行榜列表

## 暂不支持

1. Redis 排行榜缓存
2. 动态分
3. 队伍排行榜
4. 分页之外的复杂筛选
5. 比赛维度排行榜
6. 趋势图
7. 解题时间线

## 排名数据来源

排行榜只根据 `solves` 表计算。

错误提交不影响排行榜。

重复正确提交不影响排行榜。

## 排序规则

Version 0.1 使用简单稳定的排序规则：

1. 总分高的排前面。
2. 总分相同时，最后一次正确提交时间更早的排前面。
3. 如果仍然相同，用户 ID 更小的排前面。

其中：

| 指标 | 计算方式 |
| --- | --- |
| 总分 | `sum(solves.score)` |
| 解题数量 | `count(solves.id)` |
| 最后一次正确提交时间 | `max(solves.solved_at)` |

## 空排行榜规则

如果还没有任何解题记录，排行榜返回空列表。

不要返回错误。

## 返回字段

排行榜每一项至少包含：

| 字段 | 说明 |
| --- | --- |
| `rank` | 排名 |
| `user_id` | 用户 ID |
| `username` | 用户名 |
| `score` | 总分 |
| `solved_count` | 解题数量 |
| `last_solved_at` | 最后一次正确提交时间 |

## 排名编号规则

Version 0.1 先使用连续排名。

示例：

```text
第 1 名：1000 分
第 2 名：900 分
第 3 名：900 分
第 4 名：800 分
```

即使分数相同，也根据排序规则给出不同名次。

## 后端建议分层

```text
controller
└── RankingController

service
└── RankingService

repository
└── SolveRepository

dto
└── RankingItemResponse
```

## 性能说明

Version 0.1 数据量很小，实时聚合查询可以接受。

后续如果数据量增大，再考虑：

1. Redis Sorted Set
2. 排行榜快照表
3. 提交后异步更新排名
4. 比赛维度缓存
