# JVM 与线程模型

## 关注点

CTF 平台涉及 Web 请求、异步判题、容器调度和后台任务，理解 JVM 与线程模型有助于稳定运行服务。

## JVM 基础

- 堆内存：保存对象实例。
- 栈内存：保存线程调用栈和局部变量。
- 元空间：保存类元数据。
- 直接内存：常见于 NIO、网络库和部分容器客户端。

## 线程来源

- Web 容器请求线程。
- 业务线程池。
- 判题 Worker 线程。
- 消息队列消费线程。
- 定时任务线程。
- 数据库连接池和 Redis 客户端内部线程。

## 常见风险

- 请求线程中执行耗时判题逻辑。
- 线程池无界队列导致内存膨胀。
- 锁竞争导致吞吐下降。
- 阻塞 I/O 占满业务线程池。
- 线程泄漏导致服务长期运行后异常。

## 排查手段

- `jstack` 查看线程堆栈。
- `jmap` 和堆 dump 分析内存。
- GC 日志分析停顿和回收频率。
- 监控线程池活跃数、队列长度和拒绝次数。
