# Spring Bean 生命周期

## 生命周期概览

Spring Bean 从定义加载到销毁大致经历实例化、属性填充、初始化、使用和销毁几个阶段。

## 主要阶段

1. 读取 BeanDefinition。
2. 实例化 Bean。
3. 填充属性和依赖注入。
4. 执行 Aware 回调。
5. 执行 BeanPostProcessor 前置处理。
6. 执行初始化方法。
7. 执行 BeanPostProcessor 后置处理。
8. Bean 进入可使用状态。
9. 容器关闭时执行销毁逻辑。

## 平台中的关注点

- 服务启动时初始化题目缓存或系统配置。
- 判题 Worker 启动和停止时处理资源清理。
- Docker 客户端、线程池和消息队列连接的生命周期。
- 通过 BeanPostProcessor 统一增强组件行为。

## 常见问题

- 在构造函数中访问尚未注入完成的依赖。
- 初始化逻辑过重导致服务启动慢。
- Bean 销毁时未关闭线程池或连接。
- 循环依赖导致代理对象行为不符合预期。
