# 整体架构

## 背景

本文用于描述 CTF 平台的整体系统边界、核心模块和运行链路。

## 架构目标

- 支持用户注册、登录、权限控制和团队协作。
- 支持比赛创建、题目发布、提交记录、排行榜和公告管理。
- 支持判题任务异步执行，保证结果可靠回写。
- 支持容器化题目环境的创建、回收、隔离和资源限制。

## 核心模块

| 模块 | 职责 |
| --- | --- |
| 用户系统 | 账号、角色、团队、权限 |
| 比赛系统 | 比赛、题目、提交、排行榜 |
| 判题系统 | 提交队列、判题执行、结果回写 |
| 容器系统 | 动态靶机、网络隔离、生命周期管理 |
| 管理后台 | 题目管理、比赛配置、用户管理、审计 |

## 典型链路

1. 用户登录平台并进入比赛。
2. 用户查看题目，必要时申请动态容器环境。
3. 用户提交 flag 或附件题答案。
4. 比赛系统生成提交记录，并投递判题任务。
5. 判题系统异步处理任务并回写结果。
6. 排行榜根据有效提交刷新分数和排名。

## 部署视图

- Web 前端：承载用户侧和管理侧页面。
- API 服务：提供业务接口、鉴权、数据聚合。
- 判题 Worker：消费判题任务，执行校验逻辑。
- 容器调度服务：对接 Docker API 或容器集群。
- 数据库：保存用户、比赛、题目、提交、日志等数据。
- 缓存与消息队列：加速排行榜读取，解耦判题任务。
