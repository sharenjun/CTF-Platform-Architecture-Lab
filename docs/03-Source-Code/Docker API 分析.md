# Docker API 分析

## 阅读目标

理解平台如何通过 Docker API 管理动态题目环境，包括镜像、容器、网络、端口映射和资源限制。

## 核心 API

- 镜像拉取和构建。
- 容器创建。
- 容器启动、停止、删除。
- 容器状态查询。
- 容器日志读取。
- 网络创建和连接。
- 资源限制配置。

## 平台调用链

1. 用户请求启动题目环境。
2. API 服务校验权限并创建容器实例记录。
3. 容器服务调用 Docker API 创建容器。
4. 容器启动后回写访问地址和过期时间。
5. 定时任务扫描过期容器并清理。

## 关键参数

| 参数 | 说明 |
| --- | --- |
| Image | 题目镜像 |
| Env | 环境变量 |
| ExposedPorts | 暴露端口 |
| HostConfig.PortBindings | 宿主机端口映射 |
| HostConfig.Memory | 内存限制 |
| HostConfig.CpuShares | CPU 权重 |
| NetworkMode | 网络模式 |

## 安全关注点

- 禁止使用特权容器。
- 控制镜像来源。
- 限制容器可访问网络。
- 避免挂载宿主机敏感目录。
- 清理容器日志和临时文件。
