# 比赛系统

## 模块定位

比赛系统负责比赛配置、题目展示、提交记录、积分计算和排行榜。

## 核心能力

- 比赛创建、编辑、发布、归档。
- 比赛报名和参赛资格校验。
- 题目分类、分值、附件和提示管理。
- 用户提交答案。
- 判题结果接收和分数计算。
- 排行榜生成和缓存刷新。
- 公告发布和比赛通知。

## 主要实体

| 实体 | 说明 |
| --- | --- |
| Contest | 比赛配置 |
| Challenge | 题目 |
| Category | 题目分类 |
| Hint | 题目提示 |
| Submission | 提交记录 |
| ScoreRecord | 得分记录 |
| Announcement | 公告 |

## 关键规则

- 未开始的比赛不能提交答案。
- 已结束的比赛默认不再计分。
- 同一用户或团队对同一题目首次正确提交计分。
- 错误提交需要做频率限制。
- 排行榜需要兼顾准确性和读取性能。

## 状态流转

| 对象 | 状态 |
| --- | --- |
| Contest | Draft、Published、Running、Ended、Archived |
| Challenge | Draft、Online、Offline |
| Submission | Pending、Judging、Accepted、WrongAnswer、Failed |
