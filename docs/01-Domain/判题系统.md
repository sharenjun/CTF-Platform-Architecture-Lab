# 判题系统

## 模块定位

判题系统负责接收提交任务、执行判题逻辑并将结果可靠回写到比赛系统。

## 核心能力

- 创建判题任务。
- 异步消费提交记录。
- 校验 flag 或运行自定义判题脚本。
- 处理判题超时和异常。
- 幂等回写判题结果。
- 记录判题日志和错误原因。

## 主要实体

| 实体 | 说明 |
| --- | --- |
| JudgeTask | 判题任务 |
| JudgeResult | 判题结果 |
| JudgeLog | 判题日志 |
| Submission | 用户提交 |

## 判题流程

1. 比赛系统保存提交记录，状态为 `Pending`。
2. 系统投递 `JudgeTask` 到队列。
3. 判题 Worker 拉取任务并标记为 `Judging`。
4. Worker 执行答案校验或脚本判题。
5. Worker 回写 `Accepted`、`WrongAnswer` 或 `Failed`。
6. 比赛系统根据结果更新积分和排行榜。

## 并发与可靠性

- 判题任务需要具备幂等性。
- 同一提交记录只能产生一个最终判题结果。
- Worker 异常退出后任务应可重试。
- 需要限制单用户高频提交造成的队列压力。
