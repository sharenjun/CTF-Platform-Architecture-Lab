# 用户系统

## 模块定位

用户系统负责平台身份、账号资料、角色权限和团队关系，是其他业务模块的权限入口。

## 核心能力

- 用户注册、登录、登出。
- 密码加密和登录失败保护。
- 角色授权和接口权限控制。
- 团队创建、加入、退出和邀请。
- 用户资料、头像、联系方式维护。
- 管理员禁用、重置密码和审计操作。

## 主要实体

| 实体 | 说明 |
| --- | --- |
| User | 用户基础信息 |
| Role | 平台角色 |
| Permission | 权限点 |
| Team | 参赛团队 |
| TeamMember | 团队成员关系 |
| LoginLog | 登录日志 |

## 关键规则

- 用户名、邮箱或手机号必须满足唯一约束。
- 密码只保存哈希值，不保存明文。
- 普通选手不能访问管理接口。
- 团队队长可以管理团队成员，但不能越权管理其他团队。
- 禁用用户不能登录，也不能继续提交答案。

## 主要接口

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/users/me`
- `PUT /api/users/me`
- `POST /api/teams`
- `POST /api/teams/{teamId}/join`
