# 容器系统

## 模块定位

容器系统负责为动态题目创建隔离运行环境，并管理容器生命周期、访问入口和资源限制。

## 核心能力

- 创建题目容器实例。
- 分配端口、域名或访问代理。
- 查询容器运行状态。
- 延长或重置容器有效期。
- 停止并清理过期容器。
- 限制 CPU、内存、网络和文件系统权限。

## 主要实体

| 实体 | 说明 |
| --- | --- |
| ContainerImage | 题目镜像 |
| ContainerInstance | 用户容器实例 |
| PortMapping | 端口映射 |
| NetworkPolicy | 网络策略 |
| ContainerEvent | 容器事件 |

## 生命周期

1. 用户请求启动动态环境。
2. 系统校验比赛、题目和用户权限。
3. 容器系统创建实例并写入访问地址。
4. 用户访问环境完成题目。
5. 到期后系统停止并删除容器。

## 安全关注点

- 禁止容器以特权模式运行。
- 限制挂载宿主机敏感目录。
- 控制容器网络访问范围。
- 对镜像来源和构建过程做审计。
- 清理过期实例，避免资源耗尽。
