# Version 0.1：平台最小可用版本

## 目标

完成一个最小可用的 CTF 平台，支持用户注册、登录、查看题目、提交 Flag、查看排行榜。

这个版本的重点不是功能丰富，而是跑通平台的第一个业务闭环：

```text
注册 → 登录 → 查看题目 → 提交 Flag → 排行榜变化
```

只要这个闭环完成，项目就从纯文档阶段进入真实软件系统阶段。

## 功能范围

Version 0.1 只包含最核心的平台能力。

1. 用户注册
2. 用户登录
3. JWT 认证
4. 查看题目列表
5. 查看题目详情
6. 提交 Flag
7. 判断 Flag 正误
8. 记录解题记录
9. 查看排行榜

## 暂不支持

以下能力不进入 Version 0.1，避免一开始把系统做得过大。

1. 管理员后台
2. 上传题目
3. 上传附件
4. Docker 动态靶机
5. 比赛开始和结束时间
6. 动态积分
7. 队伍系统
8. Redis 缓存
9. 消息队列
10. 微服务
11. Kubernetes

这些能力会在后续版本中根据真实需求逐步加入。

## 核心业务流程

### 用户注册

用户输入用户名、邮箱和密码，系统创建用户账号。

注册成功后，用户可以使用账号登录平台。

### 用户登录

用户输入账号和密码，系统验证身份。

验证通过后，系统签发 JWT，前端后续请求需要携带这个 Token。

### 查看题目

用户登录后可以查看题目列表。

题目列表至少包含题目标题、分类、分值、解题人数和当前用户是否已解出。

用户可以进入题目详情页查看题目描述和提交入口。

### 提交 Flag

用户在题目详情页提交 Flag。

系统判断 Flag 是否正确。

如果 Flag 错误，返回错误结果，不增加分数。

如果 Flag 正确，并且用户之前没有解出该题，则记录解题记录，并影响排行榜。

### 查看排行榜

排行榜根据用户解题情况计算。

Version 0.1 可以先使用简单规则：

1. 按总分从高到低排序
2. 总分相同时，按最后一次正确提交时间从早到晚排序

## 领域对象

Version 0.1 先关注以下领域对象。

### 用户

用户是平台的基本参与者。

用户可以注册、登录、查看题目、提交 Flag、参与排行榜。

### 题目

题目是 CTF 平台的核心资源。

每道题至少包含标题、描述、分类、分值、标准 Flag、状态和创建时间。

### 提交记录

提交记录表示用户对某道题的一次 Flag 提交。

无论正确或错误，都可以记录提交行为。

### 解题记录

解题记录表示用户已经成功解出某道题。

同一个用户对同一道题只能有一条有效解题记录。

### 排行榜

排行榜是根据解题记录计算出来的结果。

Version 0.1 可以先实时查询数据库计算，不引入 Redis 缓存。

## 初始数据库表

Version 0.1 暂定需要以下数据表。

1. `users`
2. `challenges`
3. `submissions`
4. `solves`

后续会在数据库设计文档中详细定义字段、索引、约束和关系。

## 初始 API

Version 0.1 暂定需要以下 API。

### 认证

1. `POST /api/auth/register`
2. `POST /api/auth/login`

### 题目

1. `GET /api/challenges`
2. `GET /api/challenges/{id}`

### 提交

1. `POST /api/challenges/{id}/submissions`

### 排行榜

1. `GET /api/rankings`

后续会在 API 文档中详细定义请求参数、响应结构、错误码和权限要求。

## 技术范围

Version 0.1 使用尽量简单、直接、可维护的技术组合。

1. Java 21
2. Spring Boot
3. Spring MVC
4. Spring Security
5. JWT
6. PostgreSQL
7. JPA 或 MyBatis
8. Vue 3

暂时不引入 Redis、消息队列、微服务和 Kubernetes。

## 完成标准

Version 0.1 完成时，系统至少应该满足以下条件。

1. 用户可以成功注册
2. 用户可以成功登录并拿到 JWT
3. 受保护接口必须携带 JWT 才能访问
4. 用户可以查看题目列表
5. 用户可以查看题目详情
6. 用户可以提交 Flag
7. 正确 Flag 会生成解题记录
8. 同一用户重复提交同一题的正确 Flag 不会重复加分
9. 排行榜能根据解题记录展示用户排名
10. 后端接口有基本的错误处理
11. 数据库表结构清晰，有必要的唯一约束和索引
